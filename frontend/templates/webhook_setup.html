Setup Instructions
1
Navigate to Docker Hub

Go to Docker Hub and sign in to your account.

2
Select Repository

Choose the repository you want to monitor (e.g., {{ backend_info.monitoring_config.dockerhub_username or 'your-username' }}/your-repo).

3
Open Webhooks Settings

Click on the repository, go to Settings → Webhooks.

4
Add New Webhook

Click "Create Webhook" and configure:

  • Webhook name: Vulnerability Scanner
  • Webhook URL: {{ backend_url }}/webhook/dockerhub
  • Triggers: Repository push
5
Save and Test

Save the webhook and push a new image to test automatic scanning.

Pro Tips
  • Set up webhooks for all your critical repositories
  • Test webhooks after setup to ensure they work
  • Monitor the scan queue for webhook-triggered scans
  • Enable Slack notifications for real-time alerts

Troubleshooting

  • Verify the webhook URL is correct and accessible
  • Check that the webhook is configured for "Repository push" events
  • Ensure your backend service is running and reachable
  • Check the scan queue for pending items

  • Check if Trivy is properly installed and updated
  • Verify network connectivity for CVE database access
  • Monitor system resources (CPU, memory, disk space)
  • Review scan timeout settings in configuration

  • Verify Slack webhook URL is correctly configured
  • Test Slack integration with a manual scan
  • Check Slack app permissions and channel settings
  • Review application logs for notification errors
